# Insight Face Project Developments

ဤစာမျက်နှာတွင် insight_face project ၏ လက်ရှိ တိုးတက်မှုအခြေအနေများနှင့် အသုံးပြုထားသော နည်းပညာများအကြောင်းကို ဖော်ပြထားသည်။

## Developed Features

ယခုလက်ရှိတွင် အောက်ပါ features များသည် develop လုပ်ပြီးသားဖြစ်ပြီး အသုံးပြုနိုင်ပြီဖြစ်သည်။

1.  **Face Detection (မျက်နှာရှာဖွေခြင်း):** ပေးထားသော ပုံ သို့မဟုတ် video stream ထဲမှ လူမျက်နှာများကို ရှာဖွေနိုင်သည်။
2.  **Face Recognition (မျက်နှာဖြင့်မှတ်သားခြင်း):** ရှာဖွေတွေ့ရှိသော မျက်နှာများကို `data/gallery` ထဲတွင်ရှိသော လူများနှင့် တိုက်ဆိုင်စစ်ဆေးပြီး မည်သူမည်ဝါဖြစ်သည်ကို ခွဲခြားနိုင်သည်။
3.  **Liveness Detection (တကယ်လူဟုတ်မဟုတ်စစ်ဆေးခြင်း):** မျက်နှာသည် တကယ့်လူလား (live) သို့မဟုတ် ဓာတ်ပုံ၊ ဗီဒီယိုလား (spoof) ဆိုတာကို စစ်ဆေးပေးနိုင်သည်။
4.  **Face Swapping (မျက်နှာချင်းလဲလှယ်ခြင်း):** ပုံနှစ်ပုံထဲမှ မျက်နှာများကို အပြန်အလှန် လဲလှယ်နိုင်သည်။
5.  **Gender and Age Prediction (ကျား/မ နှင့် အသက်ခန့်မှန်းခြင်း):** မျက်နှာမှ ကျား/မ နှင့် အသက်ကို ခန့်မှန်းပေးနိုင်သည်။

## Models

Project တွင် `buffalo_l` အပြင် အခြားသော ONNX model များကိုလည်း အသုံးပြုထားသည်။

### အသုံးပြုထားသော Models များ

1.  **`buffalo_l`**:
    *   **အကြောင်းအရင်း:** ၎င်းသည် insightface မှ ထုတ်လုပ်ထားသော SOTA (State-of-the-art) face analysis model တစ်ခုဖြစ်ပြီး မျက်နှာရှာဖွေခြင်း၊ မှတ်သားခြင်း၊ ကျား/မ နှင့် အသက်ခန့်မှန်းခြင်းတို့အတွက် တိကျမှုမြင့်မားစွာဖြင့် လုပ်ဆောင်ပေးနိုင်သည်။
    *   **`buffalo_l` ထဲရှိ Model များ:**
        *   `det_10g.onnx`: Face Detection model ဖြစ်ပြီး မျက်နှာများကို ရှာဖွေရန် အသုံးပြုသည်။
        *   `w600k_r50.onnx`: Face Recognition model ဖြစ်ပြီး မျက်နှာတစ်ခုချင်းစီအတွက် feature vector (embedding) များကို ထုတ်ပေးသည်။ ဤ vector များကို နှိုင်းယှဉ်ခြင်းဖြင့် မျက်နှာများကို ခွဲခြားနိုင်သည်။
        *   `genderage.onnx`: Gender နှင့် Age prediction model ဖြစ်သည်။
        *   `1k3d68.onnx` နှင့် `2d106det.onnx`: မျက်နှာပေါ်ရှိ landmark (အမှတ်အသားများ) ကို ရှာဖွေရန် အသုံးပြုသည်။ ဥပမာ- မျက်လုံး၊ နှာခေါင်း၊ ပါးစပ်နေရာများ။

2.  **`2.7_80x80_MiniFASNetV2.onnx`**:
    *   **အကြောင်းအရင်း:** ဤ model သည် Liveness Detection (Face Anti-Spoofing) အတွက် အထူးပြုလုပ်ထားသော model ဖြစ်သည်။ မျက်နှာသည် တကယ့်လူအစစ်ဟုတ်မဟုတ်ကို စစ်ဆေးပေးနိုင်သောကြောင့် security ပိုင်းအတွက် အရေးကြီးသည်။

3.  **`inswapper_128.onnx`**:
    *   **အကြောင်းအရင်း:** ဤ model သည် Face Swapping ပြုလုပ်ရန်အတွက် အသုံးပြုသည်။ insightface ၏ `inswapper` model ဖြစ်ပြီး မျက်နှာများကို တိကျစွာ လဲလှယ်ပေးနိုင်သည်။

## Scripts

`scripts/` directory ထဲရှိ Python script များ၏ အလုပ်လုပ်ပုံမှာ အောက်ပါအတိုင်းဖြစ်သည်။

*   **`detect.py`**: ပေးထားသော input (image/video) မှ မျက်နှာများကို ရှာဖွေပြီး bounding box များဖြင့် ပြန်လည်ထုတ်ပေးသည်။ `buffalo_l` ထဲမှ `det_10g.onnx` ကို အသုံးပြုသည်။
*   **`recognize.py`**: `detect.py` ဖြင့် မျက်နှာများကို အရင်ရှာသည်။ ထို့နောက် `w600k_r50.onnx` model ကို အသုံးပြု၍ မျက်နှာတစ်ခုချင်းစီအတွက် embedding vector များထုတ်သည်။ ရလာသော vector များကို `data/gallery` ထဲရှိ လူများ၏ vector များနှင့် နှိုင်းယှဉ်ပြီး မည်သူဖြစ်ကြောင်း ခွဲခြားသတ်မှတ်သည်။
*   **`liveness.py`**: `2.7_80x80_MiniFASNetV2.onnx` model ကို အသုံးပြု၍ real-time video stream သို့မဟုတ် ပုံများထဲမှ မျက်နှာများ၏ liveness score ကို တွက်ချက်ပေးသည်။
*   **`swap.py`**: `inswapper_128.onnx` model ကို အသုံးပြု၍ source image နှင့် target image တို့မှ မျက်နှာများကို အပြန်အလှန် လဲလှယ်ပေးသည်။
*   **`convert_liveness_to_onnx.py`**: PyTorch model (`.pth`) ကို ONNX format (`.onnx`) သို့ ပြောင်းလဲပေးရန် အသုံးပြုသည့် utility script ဖြစ်သည်။ `Silent-Face-Anti-Spoofing` မှ `2.7_80x80_MiniFASNetV2.pth` ကို `.onnx` သို့ ပြောင်းရန် အသုံးပြုခဲ့သည်။

## Vector Storage

> Vector တွေကို Numpy array တစ်ခုနဲ့ Pickle ဖိုင်တစ်ခုနဲ့ သိမ်းဖို့ရော အဆင်ပြေပြီလား၊ဆိုတာသိချင်ပါတယ်။

**အဖြေ:** **အဆင်ပြေပါသည်။**

Face recognition မှ ရရှိလာသော embedding vector များကို NumPy array အဖြစ်သို့ ပြောင်းလဲပြီး `pickle` library ကို အသုံးပြု၍ `.pkl` သို့မဟုတ် `.npy` file format များဖြင့် သိမ်းဆည်းနိုင်ပါသည်။ ဤနည်းလမ်းသည် အောက်ပါ အားသာချက်များရှိသည်။

*   **Performance:** မျက်နှာများကို မှတ်သားရန်လိုအပ်သည့်အခါတိုင်း model ကို run ရန်မလိုတော့ဘဲ သိမ်းဆည်းထားသော vector များကို ပြန်လည် load လုပ်ပြီး တိုက်ရိုက်အသုံးပြုနိုင်သောကြောင့် အချိန်ကုန်သက်သာစေသည်။
*   **Portability:** Vector ဖိုင်များကို အလွယ်တကူ ရွှေ့ပြောင်းနိုင်ပြီး မတူညီသော script များ သို့မဟုတ် application များတွင် ပြန်လည်အသုံးပြုနိုင်သည်။

ဥပမာအားဖြင့် `data/gallery` ထဲရှိ လူများ၏ မျက်နှာဓာတ်ပုံများမှ embedding vector များကို တစ်ခါ ကြိုတင်ထုတ်ပြီး `embeddings.pkl` ကဲ့သို့သော ဖိုင်ဖြင့် သိမ်းထားနိုင်သည်။ `recognize.py` ကို run သည့်အခါ ထိုဖိုင်ကို load လုပ်ပြီး မျက်နှာများကို တိုက်ဆိုင်စစ်ဆေးနိုင်သည်။
